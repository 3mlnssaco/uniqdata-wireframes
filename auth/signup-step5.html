📋 디자이너 노트: 블록체인 앵커링

페이지 목적: 패스키-지갑 바인딩을 XRPL에 영구 기록. 법적 증거력 확보.

앵커링 5단계:
1. 챌린지 생성: 서버가 랜덤 challenge 생성
2. 패스키 서명: 사용자 생체인증 → 서명
3. TX 전송: 서명된 Payment TX를 XRPL에 제출
4. 블록 확인: ~3-5초 대기 (ledger close)
5. 완료: TX Hash 발급, DB에 저장

사용자 경험:
• "앵커링 시작" 버튼 클릭
• 생체인증 팝업 → 서명
• 각 단계가 순서대로 완료 표시
• 완료 시 TX Hash와 Explorer 링크 표시

⚙️ 기능 명세

핵심 기능:
• 패스키-지갑 바인딩 XRPL 앵커링
• 법적 증거력 있는 영구 기록 생성
• TX Hash 발급 및 저장

앵커링 5단계:
1. 챌린지 생성 (서버)
2. 패스키 서명 (생체인증)
3. Payment TX 전송 (XRPL)
4. 블록 확인 (~3-5초)
5. TX Hash 발급 완료

🔗 백엔드 호출

1. 앵커링 챌린지 요청:
POST /api/v1/xrpl/anchor/challenge
{ "wallet_address": "rN7n3...", "credential_id": "xxx" }
→ { "challenge": "random32bytes", "memo_data": "..." } 2. 서명된 TX 제출:
POST /api/v1/xrpl/anchor/submit
{ "signed_tx": "1200002280..." }
→ { "tx_hash": "A1B2C3...", "ledger_index": 12345678 } 3. 앵커링 확인:
GET /api/v1/xrpl/anchor/verify/{tx_hash}
→ { "verified": true, "timestamp": "2026-02-01T10:00:00Z" }

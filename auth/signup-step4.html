🔍 왜 패스키로 지갑을 보호하나요? ▼

PRF (Pseudo Random Function): 패스키에서만 추출 가능한 고유 암호화 키

  • XRPL 지갑 시드를 PRF 키로 암호화
  • 패스키 없이는 시드 복호화 불가
  • 서버 해킹되어도 암호화된 시드만 유출 (쓸모없음)
  • 사용자는 패스키 하나만 관리하면 됨

📋 디자이너 노트: 패스키 등록 (통합)

기존 Step 4 + 5 통합:
• 패스키 설명과 등록을 하나로 합쳐 UX 개선
• 간략한 소개 → 자세히 보기 아코디언 → 등록 버튼
• 정보 과부하 방지 + 필요시 상세 정보 제공

처리 순서:
1. XRPL 지갑 생성 (주소+시드)
2. 생체인증 + PRF 키 추출
3. 시드 암호화 (AES-256-GCM) + 서버 저장

보안 원칙:
• 평문 시드: 브라우저 밖으로 절대 전송 안 함
• PRF 키: 서버로 전송 안 함 (클라이언트만)
• 서버 저장: 암호화된 blob만

⚙️ 기능 명세

핵심 기능:
• 패스키 개념 설명 (간략 + 상세 아코디언)
• XRPL 지갑 생성 (Testnet/Mainnet)
• WebAuthn 패스키 등록 + PRF 확장
• PRF 키로 지갑 시드 암호화 (AES-256-GCM)
• 암호화된 시드 서버 저장

처리 순서:
1. XRPL 지갑 생성 → 주소 + 시드 획득
2. 서버에서 WebAuthn 등록 옵션 요청
3. navigator.credentials.create() + PRF extension
4. PRF 결과에서 32바이트 암호화 키 추출
5. AES-256-GCM으로 시드 암호화
6. credential + encrypted_seed 서버 전송

→ { classicAddress: "rN7n3...", seed: "sn3nxiW9v..." } 2. Passkey 등록 옵션:

→ { "challenge": "...", "rp": {...}, "extensions": { "prf": {} } } 3. 패스키 등록 + PRF:

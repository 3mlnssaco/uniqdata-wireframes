📋 디자이너 노트: 로그인

로그인 플로우 (3단계):
1. 이메일 + 비밀번호: 기본 인증
2. 이메일 인증 코드: 2차 인증 (6자리 코드)
3. 패스키 인증: 생체인증으로 최종 확인

또는 빠른 로그인:
• 패스키만으로 바로 로그인 가능 (이메일+비번 스킵)
• Resident Key가 있으면 이메일 입력도 필요 없음

패스키 분실 복구:
1. 이메일 + 비밀번호로 로그인 시도
2. 이메일 인증 코드 입력
3. 패스키 없음 → "새 패스키 등록" 옵션 제공
4. 새 기기에서 패스키 재등록

보안 포인트:
• 패스키 없이는 민감 작업 불가 (에스크로 등)
• 복구 시에도 3단계 인증 필요
• 지갑 시드는 새 패스키의 PRF로 재암호화

⚙️ 기능 명세

로그인 방식:
• 패스키 바로 로그인 (생체인증만)
• 이메일 + 비밀번호 → 이메일 인증 → 패스키

패스키 분실 복구:
1. 이메일 + 비밀번호 인증
2. 이메일 인증 코드 입력
3. 새 패스키 등록 옵션 제공
4. 새 PRF로 지갑 시드 재암호화

🔗 백엔드 호출

1. 패스키 인증 옵션:
POST /api/v1/auth/passkey/authenticate/options
{ "email": "researcher@example.com" } // 또는 빈 객체 (Resident Key)
→ { challenge, rpId, allowCredentials, extensions: { prf: true } } 2. 패스키 인증 완료:
POST /api/v1/auth/passkey/authenticate/complete
{ "credential": { id, type, response, clientExtensionResults } }
→ { "access_token": "jwt...", "refresh_token": "...", "user": {...} } 3. 이메일 + 비밀번호 로그인:
POST /api/v1/auth/email/login
{ "email": "...", "password": "..." }
→ { "access_token": "jwt...", "refresh_token": "...", "user": {...} }

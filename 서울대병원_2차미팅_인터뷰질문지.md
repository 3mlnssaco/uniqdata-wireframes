- ▸ 보상금을 미리 예치해두면 참여자 신뢰도가 올라갈까요?
- ▸ 비용 분해(IRB 수수료 + 참여자 보상 + 플랫폼 수수료)가 투명하게 보이는 것 — 연구비 집행 관리에 도움이 될까요?

- ▸ 포인트 단가(예: 1포인트 = 1원)를 어떻게 설정하면 참여자에게 직관적일까요?

**국제 연구 — 토큰/NFT 보상 (참고):**
- ▸ 국제 다기관 연구 시 환율/송금 문제가 있나요? 토큰 기반 정산이 이 문제를 줄일 수 있다고 보시나요?
- ▸ 해외 기관과 공동 연구 시 보상 지급의 가장 큰 애로사항은?

### 2-1. 포괄동의 + IRB 원스톱

**Q9. 포괄동의(broad consent) 모델에 대해 어떻게 생각하시나요? 한 번 동의로 다수 연구에 활용할 수 있는 구조입니다.**

> 💡 현행: 연구마다 개별 IRB + 개별 동의

**포괄동의 현황:**
- ▸ 한국에서 포괄동의가 실제 적용된 사례를 아시나요? (바이오뱅크, 코호트 연구 등)
- ▸ 생명윤리법에서 포괄동의의 현재 법적 지위는 어떤가요? (허용? 제한적 허용? 불명확?)
- ▸ 서울대병원에서 포괄동의 기반 연구가 진행된 적이 있나요?

**자가입력 데이터 + 동의:**
- ▸ 환자가 앱에서 직접 입력한 데이터(PRO, Patient-Reported Outcome)를 연구에 쓰는 경우 — IRB에서 별도 심의가 필요한가요?
- ▸ 개인이 자기 데이터를 동의 하에 연구 목적으로 제공하는 것 — 기존 IRB 프레임워크에 맞나요? 아니면 새로운 규정이 필요한가요?
- ▸ 동의 범위를 세분화(예: "혈당 데이터는 OK, 위치 데이터는 NO")할 수 있다면 IRB 심의가 더 수월해질까요?
- ▸ 동의 철회(opt-out) 시 이미 수집된 데이터 처리는 어떻게 해야 하나요?

**다수 연구 활용:**
- ▸ 한 번 동의로 여러 연구에 활용하는 구조 — 연구자 입장에서 가장 큰 장점과 우려는?
- ▸ 동의 갱신 주기가 필요하다면 적절한 기간은? (1년? 2년? 연구 종료 시?)

`[메모]`

### 2-2. Federated Learning / 모델 공유

**Q10. 원광대 협력 시 코드를 들고 가서 현지에서 돌리는 방식을 자동화한 플랫폼이 있다면 쓸 의향이 있으신가요?**

> 💡 데이터는 안 움직이고 모델이 움직이는 방식

**현행 다기관 연구 프로세스:**
- ▸ 현재 다기관 연구의 가장 큰 애로사항은? (IRB 각각? 데이터 형식 불일치? 보안? 물리적 이동?)
- ▸ "코드를 USB에 담아 가서 현지에서 돌린다" — 이 방식의 실제 빈도와 소요 시간은?
- ▸ 타 기관 데이터를 분석하러 갈 때 보안 절차(접근 승인 등)는 어떻게 되나요?
- ▸ 다기관 연구 결과를 합치는(meta-analysis, pooled analysis) 과정에서 겪는 어려움은?

**연합학습(Federated Learning) 가능성:**
- ▸ 연합학습(FL)을 실제 적용해 보신 적 있으신가요? 관심을 갖고 계시다면 어떤 연구 주제에?
- ▸ 도커 이미지로 모델을 패키징해서 보내는 방식 — IT팀 관점에서 기술적으로 무리가 없을까요?
- ▸ 연합학습 결과의 신뢰성 검증은 어떻게 해야 한다고 보시나요?
- ▸ 서울대병원 IT 인프라에서 외부 도커 이미지를 실행하는 것 — 보안 정책상 가능한가요?
- ▸ 해외 기관과 FL을 한다면, 어떤 나라/기관이 가장 현실적인 파트너일까요?

`[메모]`

### 2-3. CDM/OMOP 변환

**Q11. EHR 데이터를 자동으로 CDM으로 변환해주는 서비스가 있다면 수요가 있을까요?**

> 💡 1차 피드백: OMOP CDM 변환이 몇십 년째 진행 중

**CDM 변환 현황:**
- ▸ 서울대병원의 CDM 변환 진행 상황은? (어느 단계? 완료율?)
- ▸ CDM 변환에서 가장 어려운 부분은 구체적으로? (매핑 테이블? 비정형 데이터? 용어 불일치?)
- ▸ OMOP CDM 외에 다른 CDM 표준(i2b2, PCORnet, CDISC 등)도 사용하시나요?
- ▸ CDM 변환 작업을 누가 주로 담당하나요? (전담 팀? 연구자 직접? 외주?)

**변환 품질 & 자동화:**
- ▸ 변환 품질 검증은 어떻게 하시나요? (ACHILLES? DQD? 수작업 검토?)
- ▸ AI 기반 자동 매핑(예: 진료코드 → OMOP concept_id 자동 매핑)이 가능하다면 신뢰할 수 있을까요?
- ▸ 변환 오류가 발생했을 때 가장 치명적인 영역은? (약물? 진단? 검사결과?)
- ▸ 자동 변환 도구가 있다면 어떤 기능이 가장 필요한가요? (매핑 제안? 검증 리포트? 일괄 변환?)

**다기관 CDM:**
- ▸ 다른 병원들(연세, 삼성, 아산 등)도 비슷한 어려움을 겪고 있나요?
- ▸ 병원 간 CDM 호환이 안 되는 주요 원인은?
- ▸ OHDSI 한국 챕터 활동에 참여하고 계시나요?

## Part 3. 협력 가능성 탐색 (10분)

**Q12. 연구실에서 현재 가장 시간이 많이 드는 반복 작업이 있다면?**

> 💡 자동화 가능한 반복 작업 식별 → 우리 플랫폼이 해결할 수 있는 페인 포인트 발굴

**연구 워크플로우:**
- ▸ 데이터 전처리, 통계 분석, 문서 작성 중 가장 번거로운 것은?
- ▸ 하나의 연구에서 데이터 수집 → 전처리 → 분석 → 논문 작성까지 보통 얼마나 걸리나요?
- ▸ 논문 작성 시 figure/table 생성에 시간이 많이 드나요? 자동화된다면 도움이 될까요?
- ▸ 연구 보고서(중간보고, 최종보고) 작성도 AI가 도와주면 도움이 될까요?

**IT/인프라 관점:**
- ▸ IT팀 관점에서 외부 SaaS 플랫폼 도입 시 가장 신경 쓰는 부분은? (보안 인증? 데이터 위치? SLA?)
- ▸ 서울대병원 내부 시스템과 외부 플랫폼 간 연동(API 등)이 기술적으로 가능한가요?
- ▸ 현재 연구 데이터를 클라우드에 저장하는 것이 허용되나요? (온프레미스 의무?)
- ▸ ISMS, HIPAA, PIPA 등 보안 인증 요건이 있다면 어떤 것들인가요?

- ▸ 논문 리뷰어가 외부 데이터를 요구하는 맥락은? (외적 타당성? 재현성? 표본 크기?)
- ▸ 현재 외부 데이터를 구하기 위해 어떤 경로를 쓰시나요? (공공 DB? 기관 간 MOU? 직접 수집?)
- ▸ 익명화된 데이터를 접근할 수 있다면 어떤 연구 주제에 바로 쓸 수 있을까요?

**데이터 마켓플레이스:**
- ▸ 데이터를 "구매"하는 개념에 대해 어떻게 생각하시나요? (연구비로 집행 가능?)
- ▸ 데이터 품질 보증(검증된 데이터만 등록)이 된다면 비용을 지불할 의향이 있으신가요?
- ▸ 자신의 연구 데이터를 다른 연구자에게 제공(판매/공유)하는 것은 가능한가요? IRB 제약은?
- ▸ 데이터 접근 후 사용 추적(audit trail)이 필요한가요?

## 3. 필요한 추가 요소

### 3.1 참여자별 테이블 뷰

**현재:** 날짜별 뷰만 존재 (1월 1주차, 1월 2주차...)
**필요:** 참여자별 뷰 (P001, P002... 순으로 개인별 시계열)

**구조:**
```
┌─────────┬───────┬─────────┬─────────┬─────────┬─────────┐
│ 참여자   │ 기간   │ 심박수   │ 혈당     │ 복약    │ 기분    │
├─────────┼───────┼─────────┼─────────┼─────────┼─────────┤
│ P001    │ 1/1   │ 72      │ 118     │ O       │ 4       │
│ (김OO)  │ 1/2   │ 74      │ 115     │ O       │ 3       │
│         │ 1/3   │ 71      │ 120     │ O       │ 4       │
│         │ ...   │ ...     │ ...     │ ...     │ ...     │
├─────────┼───────┼─────────┼─────────┼─────────┼─────────┤
│ P002    │ 1/1   │ 68      │ 125     │ X       │ 2       │
│ (이OO)  │ 1/2   │ 70      │ 130     │ O       │ 3       │
└─────────┴───────┴─────────┴─────────┴─────────┴─────────┘
```

**정렬/필터:**
- 참여자 ID/로컬이름으로 검색
- 가나다순/등록순/최근활동순 정렬
- 특정 참여자만 필터

### 3.2 프로젝트 지갑 구조

**변경 전:** 개인 지갑만 존재
**변경 후:** 개인 지갑 + 프로젝트별 지갑 이원화

#### 지갑 구조
```
개인 지갑 (연구자 소유)
├── 키: 사용자 보유
├── 용도: 외부 입출금, 프로젝트 지갑 충전
└── 위치: billing.html

프로젝트 지갑 (연구별)
├── 키: 서버 보유
├── 용도: 연구 예산, 참여자 정산
├── 생성: create-step5.html (예산 탭)에서 자동 생성
└── 위치: 개요 페이지에 표시
```

#### 개요 페이지에 추가할 프로젝트 지갑 정보
```
┌─────────────────────────────────────────────┐
│ 프로젝트 지갑                    rPrj...1A2 │
├─────────────────────────────────────────────┤
│ 잔액          8,500 XRP                     │
│ 예산          10,000 XRP                    │
│ 지출          1,500 XRP (15%)               │
│ ─────────────────────────────────────────── │
│ [개인지갑에서 이체]                          │
└─────────────────────────────────────────────┘
```

#### 이체 팝업 (개인 → 프로젝트)
```
┌─────────────────────────────────────────────┐
│ 프로젝트 지갑 충전                     [X]  │
├─────────────────────────────────────────────┤
│ 출발 지갑:  개인 지갑 (45,000 XRP)          │
│ 도착 지갑:  당뇨병 복약 연구 (8,500 XRP)    │
│                                             │
│ 이체 금액:  [          ] XRP                │
│                                             │
│ ⚠️ 프로젝트 예산(10,000 XRP) 대비            │
│    현재 잔액이 부족합니다 (-1,500 XRP)       │
│                                             │
│         [취소]  [이체하기]                   │
└─────────────────────────────────────────────┘
```

### 3.4 데이터 테이블 표현 통일

**변경 전:** 스파크라인, 도트, 히트맵 등 혼재
**변경 후:** 텍스트 기반 엑셀 형식

```
┌─────────┬───────┬─────────┬─────────┬─────────┬─────────┬─────────┐
│ 참여자   │ 로컬명 │ 심박수   │ 혈당     │ 복약    │ 수면    │ 기분    │
├─────────┼───────┼─────────┼─────────┼─────────┼─────────┼─────────┤
│ P001    │ 김OO  │ avg 72  │ avg 118 │ 7/7     │ 6.8h    │ 3.4     │
│ P002    │ 이OO  │ avg 85  │ avg 145 │ 5/7     │ 5.2h    │ 2.8     │
│ P003    │       │ avg 68  │ avg 110 │ 7/7     │ 7.1h    │ 4.0     │
└─────────┴───────┴─────────┴─────────┴─────────┴─────────┴─────────┘
```

**컬럼 설명:**
- 심박수: 평균값 (bpm)
- 혈당: 평균값 (mg/dL)
- 복약: O/X 횟수 (예: 7/7)
- 수면: 평균 시간
- 기분: 평균 점수 (1-5)

**확장 가능:** 오른쪽 스크롤로 추가 컬럼

### 3.5 홈 카드 개선

**변경 전:**
- 참여자 버튼 → 개요 이동
- 데이터 버튼 → 개요 이동 (중복)

**변경 후:**
- 하나의 버튼으로 통합 → 개요 이동

**7일 데이터 수집 트렌드:**
- 현재: 7일 고정 그래프
- 변경: 전체 기간 응답률 그래프 + 기간 필터
- 목적: 이탈 추이 파악 → 참여자 추가 모집 액션

## 4. 생성 플로우 변경

**변경 전:**
```
Step 1: 연구 설계
Step 2: 설계 확인
Step 3: 데이터 수집
Step 4: AI 도구 선택  ← 삭제
Step 5: 예산
Step 6: 완료
```

**변경 후:**
```
Step 1: 연구 설계
Step 2: 설계 확인
Step 3: 데이터 수집
Step 4: 예산 (프로젝트 지갑 자동 생성)
Step 5: 완료
```

**Step 4 (예산) 추가 기능:**
- 예산 입력 시 프로젝트 지갑 자동 생성
- 프로젝트 지갑 잔액 < 예산이면 경고 표시
- 개인 지갑 → 프로젝트 지갑 이체 팝업 제공

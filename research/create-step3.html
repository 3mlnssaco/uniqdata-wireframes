지표명 단위 CDM Domain 데이터 유형 우선순위 시각화
■ 체중 kg measurement numeric Primary numeric_line
■ SBP mmHg measurement numeric Secondary numeric_line
■ TIR % device_exposure proportion Secondary severity_zone
■ 피부상태 점 observation categorical Exploratory image_gallery
■ 순응도 % observation numeric Operational numeric_line
시각화는 domain × value_type에 따라 자동 결정됩니다: numeric → 라인차트  |  proportion → 스택바  |  categorical → 갤러리  |  binary → 도트맵

📁 과거 데이터만 구매
참여자의 기존 건강 데이터(EMR, CDM)를 구매하여 분석합니다.
후향적 분석 CDM/EMR 즉시 분석 가능
📱 앞으로 데이터 수집
참여자를 모집하고 앞으로 새 데이터를 수집합니다. 웨어러블, 앱 기록, 방문 측정 등.
전향적 연구 실시간 모니터링 실험군/대조군
🔄 과거 + 미래 동시
과거 데이터가 있는 참여자한테서 과거 기록 + 앞으로의 데이터를 함께 수집합니다.
양방향 코호트 풍부한 데이터 기저치 확보
📁 과거 데이터 구매: 이미 존재하는 CDM/EMR 데이터를 분석합니다. 모집 없이 바로 분석 시작 가능. 후향적 코호트, Case-control, CDM 네트워크 분석에 적합합니다.
📱 미래 데이터 수집: 참여자를 새로 모집하고 데이터를 수집합니다. RCT, 전향적 코호트, 단일군 연구에 적합합니다. 타임라인에 '오늘' 마커가 표시됩니다.
🔄 양방향 수집: 과거 데이터 보유자를 대상으로 기존 기록(기저치)과 새 데이터를 동시에 확보합니다. 기저치 대비 변화량 분석이 가능하고, 더 풍부한 데이터셋을 구축할 수 있습니다.

🤖 패시브 데이터 AI 추천 근거

AI 분석
연구 설계(RCT, 저탄수화물 식단, CGM 기반 혈당 변동)를 분석하여 4개 패시브 데이터 항목을 추천합니다. 유사 RCT 41건의 데이터 수집 패턴을 참고했습니다.
📈
CGM (연속혈당) — 필수
MAGE 계산에 최소 14일 연속 측정 필요. 5분 간격 데이터로 혈당 변동성(MAGE, CV%, TIR)을 정밀 측정합니다.
유사 RCT 41건 중 38건(93%)이 CGM을 1차 데이터로 사용
❤️
심박수 — 권장
저혈당 에피소드 시 심박수 변화 감지. 야간 저혈당 탐지 보조 지표로 활용 가능합니다.
식이 개입 RCT의 67%가 심박수를 보조 지표로 수집
🚶
활동량/걸음수 — 권장
신체 활동은 혈당에 직접 영향. 식단 효과를 분리하기 위한 통제 변수로 필수적입니다.
운동량 미통제 시 MAGE 효과크기 과대추정 위험 (bias ~15%)
😴
수면 — 권장
수면 품질/시간은 인슐린 저항성과 직결. Dawn phenomenon(새벽 현상) 분석에 수면 데이터가 필요합니다.
수면-혈당 상관 r=0.42 (Park et al. 2024)
참고: 체온, 위치/이동 데이터는 본 연구 설계에서 직접적 분석 가치가 낮아 선택 사항으로 분류했습니다. 필요 시 탐색적 분석용으로 추가 가능합니다.

🤖 능동 데이터 AI 추천 근거

AI 분석
능동 데이터는 참여자 부담과 직결됩니다. 3개 필수 항목을 추천하며, EMA 빈도는 완료율을 고려하여 최적화했습니다.
🍽️
식단 기록 — 필수 (3회/일)
저탄수화물 식단 순응도를 직접 측정하는 핵심 변수. 매 식사 후 탄수화물 섭취량을 기록합니다. 사진 촬영 + AI 분석으로 자동 추정하면 입력 시간을 70% 단축 가능합니다.
3회/일 기록 시 순응도 측정 정확도 92% (vs 1회/일 64%)
💊
복약 기록 — 필수 (복용 시)
혈당 강하제/인슐린 사용 여부는 혈당 변동의 주요 교란변수. 복약 시간과 용량을 기록하여 식단 효과를 분리합니다.
복약 미기록 시 MAGE 효과크기 해석 불가 (교란 위험)
😊
기분/컨디션 — 권장 (2회/일)
식단 개입의 삶의 질 영향을 2차 결과변수로 측정. 저탄수화물 식단 초기 피로감/짜증 등 부작용 모니터링에도 활용됩니다.
EMA 2회/일(아침/저녁)이 부담 대비 데이터 품질 최적
참여자 부담 분석: 현재 설정(식단 3회 + 복약 수시 + 기분 2회)으로 하루 약 9회 입력, 예상 소요 시간 12분/일. 완료율 유지를 위해 운동/수분 기록은 선택 사항으로 분류했습니다.

🧬 CDM 매핑 가이드

OHDSI CDM v5.4
수집 지표는 OHDSI CDM 표준 domain에 매핑됩니다. 이를 통해 다기관 CDM 네트워크(FEEDER-NET 등)와의 데이터 호환성을 확보합니다.
CDM Domain → 데이터 유형 → 자동 시각화 매트릭스
CDM Domain 저장 대상 numeric proportion categorical binary
measurement 수치 검사/측정 numeric_line severity_zone — —
observation 비수치 관찰 numeric_line — image_gallery binary_strip
device_exposure 기기 데이터 numeric_line severity_zone — —
Concept Hint: 각 지표에 concept_hint(예: body_weight, systolic_bp)가 부여되어, 추후 실제 CDM 연동 시 LOINC/SNOMED 코드와 자동 매핑됩니다.

⚙️ 기능 명세: Step 3 - 데이터 수집

핵심 기능:
• 수집 데이터 항목 선택 (토글)
• 4개 카테고리: 패시브/능동(EMA)/맥락/바이오마커
• EMA 빈도 설정 (하루 N회)
• 참여자 부담 점수 실시간 계산

데이터 민감도 레벨 (L1-L4):
• L1 (Public): app_id, 타임스탬프 - At Rest 암호화
• L2 (Internal): 연령대, 성별 - At Rest 암호화
• L3 (Confidential): 이메일, 전화 - At Rest + 필드 암호화
• L4 (Restricted): 진단코드, 처방, 바이오마커 - At Rest + 필드 암호화 + 접근 로그

동의 스코프 (Consent Scope):
• 형식: {domain}:{metric}:{permission}:{range}
• 예시: health:heart_rate:read:30d
• 예시: med:medication:write:7d
• 수집 항목별로 필요한 동의 스코프 자동 생성

AI 어시스턴트:
• 연구 목적 기반 추천 항목 표시
• "추천 적용" 버튼으로 일괄 선택
• 인과 연결 예상 설명

📋 디자이너 노트: Step 5 - IRB 제출 + 에스크로 예치

두 가지 경로 (Two-Path) 로직:
• 면제 경로 (Path A): 에스크로 예치 통과 → 바로 Step 6 모집 설정
• 심의 경로 (Path B): IRB 제출 + 에스크로 예치 → 심의 대기 → 승인 알림 → Step 6 활성화

에스크로 예치 전략:
• IRB 신청비 + 참여자 보상 에스크로 동시 예치 (연구자 편의)
• IRB 승인 시 바로 모집 가능하도록 에스크로를 미리 예치
• IRB 신청비 (400 XRP)는 제출 즉시 기관 지갑으로 전송
• 참여자 보상 (6,400 XRP)은 에스크로에 잠금 (조건부 해제)

IRB 심의 대기 상태:
• 연구 상태: "irb_pending" (심의 대기 중)
• 에스크로 상태: "funded" (예치 완료)
• IRB 승인 시 자동으로 "recruiting" 상태 전환 가능
• IRB 수정 요청 시 연구자에게 알림 → 수정 후 재제출

UI 설계 의도:
• 이 연구는 RCT이므로 Path B(심의)가 자동 선택됨
• Path A는 비활성으로 표시하여 면제 불가 사유를 명확히 전달
• 플로우 다이어그램으로 심의 과정의 분기를 시각화
• 에스크로 금액 breakdown을 보라색 카드로 강조 표시

⚙️ 기능 명세

핵심 기능:
• 지갑 잔액 확인 및 충전
• 연구 설정 전체 요약 (수정 링크 포함)
• IRB 상태 판별 (면제/정규심의)
• 에스크로 금액 산출 및 표시
• 두 가지 경로 자동 선택 UI
• 동의 체크리스트 확인

에스크로 처리:
• EscrowCreate 트랜잭션 생성 (참여자 보상)
• Payment 트랜잭션 생성 (IRB 신청비 즉시 전송)
• 조건부 해제 설정 (IRB 승인 + 참여 완료)
• TX Hash 저장 및 상태 추적

IRB 경로 판별 로직:
• RCT → 정규심의 필수
• 건강 데이터 (L3/L4) 수집 → 정규심의 필수
• 익명 설문 + L1/L2 데이터만 → 면제 가능
• 기존 데이터 2차 분석 → 면제 가능

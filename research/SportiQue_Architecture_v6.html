📐 분리 원칙 (한 줄 규칙)
"환자의 몸에서 나온 데이터, 환자의 지갑, 환자의 동의는 Core. 그 외 연구/기관 워크플로우는 UniQLab Backend."

🟢 Core (FastAPI:8000) = Core Team → 사용자 인증 + 건강 데이터 + XRPL + 동의 관리
🔵 UniQLab Backend (Spring Boot:8080) = 백엔드팀 → 연구 프로젝트 + IRB + 정산 + 기관 관리
⚠️ 경계 판단 질문: "이 데이터가 유출되면 환자가 직접 피해를 보는가?" → Yes면 Core, No면 UniQLab Backend

📐 서버 분리 Decision Matrix
판단 기준 🟢 Core (FastAPI:8000) 🔵 UniQLab Backend (Spring:8080)
데이터 민감도 민감정보 (건강데이터, 개인키, 동의 원본) 비민감 업무정보 (프로젝트, 정산, IRB 메타)
규제 적용 PIPA 민감정보 처리자, 생명윤리법 동의 관리 일반 사업자 정보 처리
블록체인 접근 XRPL 직접 R/W (지갑, 에스크로, 앵커링) XRPL 접근 불가 (Core API 경유만)
인증 책임 Passkey/PRF 처리, JWT 발급, JWKS 제공 JWT 검증만 (JWKS 공개키로, 발급 안 함)
장애 격리 UniQLab Backend 장애 시에도 환자 데이터 안전 Core 장애 시 IRB/정산 불가 (데이터 안전)
팀 배치 Core Team (보안 클리어런스 필요) 백엔드팀 (일반 개발자)
기술 스택 Python 3.11, FastAPI, SQLAlchemy, xrpl-py, python-fido2 Kotlin, Spring JPA, Claude API

🏗️ 서버별 상세 역할
🟢 uniqdata-core
FastAPI (Python) · Port 8000(default) · Core Team 담당
  • Auth (Passkey/PRF)
    WebAuthn Level 3, PRF 키 파생, JWT 발급/갱신, JWKS 엔드포인트 제공
  • Data Input ⭐
    건강 데이터 수신 → 검증 → AES-256-GCM 암호화 → uniqdata_db.data_points 저장 → XRPL 앵커링
  • Consent ⭐
    동의서 생성/서명(ed25519)/범위 관리/철회 + XRPL CONSENT_GRANT/REVOKE 앵커링
  • XRPL / UQD
    EscrowCreate/Finish/Cancel, UQD 관리, DATA_INPUT/DATA_ACCESS 앵커링
  • Notification
    푸시 알림 통합 관리 (Core + UniQLab Backend 요청 모두)
  • Internal API
    실코드 기준 13개 엔드포인트: /api/v1/internal/matching/*, /tier1/*, /token/*, /escrow/*, /health
📊 uniqlab-backend
Spring Boot (Kotlin) · Port 8080 · 백엔드팀 담당
  • Research ⭐
    프로젝트 CRUD, 마일스톤 추적, 참여자 매칭, 적격 기준, 데이터 요청
  • IRB ⭐
    제출/심의/문서 관리/워크플로우 히스토리, AI 문서 생성(Claude API)
  • Settlement ⭐
    보상 계산, settlement_batches/items 생성 → Core에 EscrowFinish 요청
  • UniQLab
    기관 등록/관리, 구성원 관리, OWNER/PI/IRB_MEMBER/STAFF 역할
  • Synthetic/FL (Phase 2)
    합성데이터셋 생성, FL 라운드 관리
  • Core Push 수신
    Core → /api/uniqlab/project/{id}/participant, /dataset(라이브), /dataset/batch, participant DELETE, /settlement(목표)
🟢 uniqdata_db — 15 Tables (Core 전용)
총 테이블: 15
1년 예상: ~2.3M 레코드
핵심: data_points ~500K/년
스토리지: PostgreSQL 15 · 2GB
#테이블설명주요 컬럼예상/년
1users사용자 원장 (인증/신원)id, email, role, kyc_level, status~10K
2passkey_credentialsFIDO2 Passkey 크레덴셜credential_id, public_key, sign_count, prf_supported, prf_salt~15K
3user_profiles개인 건강 프로필 (암호화)birth_date, gender, blood_type, medications, chronic_conditions~10K
4user_walletsXRPL 지갑 (주소만, 시드 없음)xrpl_address, xrpl_public_key, wallet_type, derivation_method~12K
5wallet_recovery_configs키 복구 설정 (SSS/Social)recovery_method, config(JSON)~5K
6data_points ⭐건강데이터 (핵심 풀)category, value(AES-256-GCM), source, data_hash, xrpl_tx_hash~500K
7data_point_tags데이터 태깅/검색tag_key, tag_value~1M
8data_access_logs접근 감사추적accessor_id, project_id, irb_number, pseudonymized, xrpl_tx_hash~50K
9consents ⭐동의 원장consent_type, status, signature_hash, grant_tx_hash, project_id~30K
10consent_scopes동의 범위 상세scope_type (DATA_CATEGORY|PURPOSE|RECIPIENT|DURATION|GEOGRAPHY)~90K
11consent_audit_trail동의 변경 이력action, actor_id, changes(JSON), xrpl_tx_hash~60K
12xrpl_transactionsXRPL tx 기록tx_hash, tx_type, memo_type, memo_data(JSON), ledger_index~200K
13notifications알림type, title, body, channel, read~300K
14notification_preferences알림 설정push_enabled, email_enabled, health_alerts, quiet_hours~10K
15refresh_tokensJWT 리프레시 토큰token_hash, device_info, expires_at, revoked~30K
CREATE TABLE users ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), email VARCHAR(255) UNIQUE NOT NULL, phone VARCHAR(20), display_name VARCHAR(100), role VARCHAR(20) NOT NULL DEFAULT 'PATIENT', -- PATIENT | RESEARCHER | IRB_MEMBER | ADMIN kyc_level SMALLINT NOT NULL DEFAULT 0, -- 0:미인증 1:이메일 2:전화 3:신분증 status VARCHAR(20) NOT NULL DEFAULT 'ACTIVE', last_login_at TIMESTAMP, created_at TIMESTAMP NOT NULL DEFAULT NOW(), updated_at TIMESTAMP NOT NULL DEFAULT NOW() ); CREATE TABLE passkey_credentials ( id UUID PRIMARY KEY, user_id UUID NOT NULL REFERENCES users(id), credential_id TEXT NOT NULL UNIQUE, public_key TEXT NOT NULL, -- COSE 공개키 sign_count BIGINT DEFAULT 0, device_type VARCHAR(30), -- 'platform'|'cross-platform' device_name VARCHAR(100), prf_supported BOOLEAN DEFAULT FALSE, prf_salt BYTEA, -- PRF용 salt (32바이트) created_at TIMESTAMP NOT NULL DEFAULT NOW() ); CREATE TABLE user_profiles ( id UUID PRIMARY KEY, user_id UUID NOT NULL UNIQUE REFERENCES users(id), birth_date DATE, -- 암호화 저장 gender VARCHAR(10), height_cm DECIMAL(5,1), weight_kg DECIMAL(5,1), blood_type VARCHAR(5), chronic_conditions TEXT[], medications JSONB, encrypted_fields JSONB, created_at TIMESTAMP NOT NULL DEFAULT NOW() );
CREATE TABLE data_points ( id UUID PRIMARY KEY, user_id UUID NOT NULL REFERENCES users(id), category VARCHAR(30) NOT NULL, -- VITALS | LAB_RESULT | MEDICATION | SYMPTOM | -- ACTIVITY | SLEEP | DIET | MENTAL_HEALTH | -- WEARABLE | MY_HEALTHWAY sub_category VARCHAR(50), -- "blood_pressure", "heart_rate", "glucose" value JSONB NOT NULL, -- AES-256-GCM 암호화된 데이터 value_encrypted BOOLEAN DEFAULT TRUE, source VARCHAR(30) NOT NULL, -- MANUAL | WEARABLE | MY_HEALTHWAY | EMR_IMPORT | FHIR measured_at TIMESTAMP NOT NULL, data_hash VARCHAR(64) NOT NULL, -- SHA-256(raw + ts) xrpl_tx_hash VARCHAR(64), -- XRPL 앵커링 tx xrpl_anchored BOOLEAN DEFAULT FALSE, quality_score DECIMAL(3,2), created_at TIMESTAMP NOT NULL DEFAULT NOW() ); CREATE INDEX idx_dp_user_cat ON data_points(user_id, category); CREATE INDEX idx_dp_user_time ON data_points(user_id, measured_at DESC); CREATE INDEX idx_dp_hash ON data_points(data_hash); CREATE TABLE data_point_tags ( id UUID PRIMARY KEY, data_point_id UUID REFERENCES data_points(id), tag_key VARCHAR(50), tag_value VARCHAR(200) ); CREATE TABLE data_access_logs ( id UUID PRIMARY KEY, data_point_ids UUID[] NOT NULL, accessor_type VARCHAR(20), -- SELF | RESEARCHER | SYSTEM | AI accessor_id UUID, project_id UUID, -- uniqdata_db.projects.id irb_number VARCHAR(50), consent_id UUID REFERENCES consents(id), access_purpose VARCHAR(50), record_count INT NOT NULL, pseudonymized BOOLEAN DEFAULT FALSE, xrpl_tx_hash VARCHAR(64), accessed_at TIMESTAMP DEFAULT NOW() );
CREATE TABLE consents ( id UUID PRIMARY KEY, user_id UUID NOT NULL REFERENCES users(id), consent_type VARCHAR(30) NOT NULL, -- SERVICE_TERMS | PRIVACY_POLICY | RESEARCH_PARTICIPATION | -- DATA_SHARING | SENSITIVE_DATA | MARKETING status VARCHAR(20) DEFAULT 'ACTIVE', -- PENDING | ACTIVE | REVOKED | EXPIRED signature_hash VARCHAR(64), -- ed25519 서명 해시 signature_method VARCHAR(30), -- PASSKEY_PRF | E_SIGNATURE | MANUAL grant_tx_hash VARCHAR(64), -- CONSENT_GRANT XRPL tx revoke_tx_hash VARCHAR(64), -- CONSENT_REVOKE XRPL tx project_id UUID, -- UniQLab Backend에서 전달받은 프로젝트 ID irb_number VARCHAR(50), version INT DEFAULT 1, created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE consent_scopes ( id UUID PRIMARY KEY, consent_id UUID REFERENCES consents(id), scope_type VARCHAR(30), -- DATA_CATEGORY | PURPOSE | RECIPIENT | DURATION | GEOGRAPHY scope_key VARCHAR(50), -- "vitals", "drug_research", "institution_id:xxxx" scope_value TEXT ); CREATE TABLE consent_audit_trail ( id UUID PRIMARY KEY, consent_id UUID REFERENCES consents(id), action VARCHAR(20), -- CREATED | SIGNED | MODIFIED | REVOKED actor_id UUID, changes JSONB, xrpl_tx_hash VARCHAR(64), created_at TIMESTAMP DEFAULT NOW() );
CREATE TABLE user_wallets ( id UUID PRIMARY KEY, user_id UUID REFERENCES users(id), xrpl_address VARCHAR(60) UNIQUE, -- rXXXXX... xrpl_public_key VARCHAR(130), -- ed25519 공개키 (hex) wallet_type VARCHAR(20) DEFAULT 'USER', -- USER | SERVICE | ESCROW derivation_method VARCHAR(20) DEFAULT 'PASSKEY_PRF', uqd_balance DECIMAL(18,6) DEFAULT 0, xrp_balance DECIMAL(18,6) DEFAULT 0, status VARCHAR(20) DEFAULT 'ACTIVE' -- ⚠️ private_key, seed는 절대 저장하지 않음! ); CREATE TABLE wallet_recovery_configs ( id UUID PRIMARY KEY, user_id UUID REFERENCES users(id), recovery_method VARCHAR(30), -- SSS | SOCIAL | HSM_BACKUP config JSONB, -- {threshold, shares, guardians, ...} created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE xrpl_transactions ( id UUID PRIMARY KEY, tx_hash VARCHAR(64) UNIQUE, tx_type VARCHAR(30), -- DATA_INPUT | DATA_ACCESS | CONSENT_GRANT | CONSENT_REVOKE | -- ESCROW_CREATE | ESCROW_FINISH | ESCROW_CANCEL | -- SYNTH_SALE | FL_ROUND | UQD_TRANSFER from_address VARCHAR(60), to_address VARCHAR(60), amount DECIMAL(18,6), currency VARCHAR(10), -- XRP | UQD memo_type VARCHAR(30), memo_data JSONB, ledger_index BIGINT, status VARCHAR(20), -- SUBMITTED | VALIDATED | FAILED related_user_id UUID, related_consent_id UUID, created_at TIMESTAMP DEFAULT NOW() );
CREATE TABLE notifications ( id UUID PRIMARY KEY, user_id UUID REFERENCES users(id), type VARCHAR(50), title VARCHAR(200), body TEXT, channel VARCHAR(20), -- PUSH | EMAIL | IN_APP read BOOLEAN DEFAULT FALSE, metadata JSONB, created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE notification_preferences ( id UUID PRIMARY KEY, user_id UUID UNIQUE REFERENCES users(id), push_enabled BOOLEAN DEFAULT TRUE, email_enabled BOOLEAN DEFAULT TRUE, health_alerts BOOLEAN DEFAULT TRUE, research_updates BOOLEAN DEFAULT TRUE, quiet_hours_start TIME, quiet_hours_end TIME ); CREATE TABLE refresh_tokens ( id UUID PRIMARY KEY, user_id UUID REFERENCES users(id), token_hash VARCHAR(64) UNIQUE, device_info JSONB, expires_at TIMESTAMP NOT NULL, revoked BOOLEAN DEFAULT FALSE, created_at TIMESTAMP DEFAULT NOW() );

CREATE TABLE institutions ( id UUID PRIMARY KEY, name VARCHAR(200) NOT NULL, institution_type VARCHAR(30), -- HOSPITAL | PHARMA | CRO | UNIVERSITY | INSURANCE | GOVERNMENT business_number VARCHAR(20), irb_committee_id UUID, status VARCHAR(20) DEFAULT 'ACTIVE', created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE institution_members ( id UUID PRIMARY KEY, institution_id UUID REFERENCES institutions(id), user_id UUID NOT NULL, -- uniqdata_db.users.id role VARCHAR(20), -- OWNER | PI | IRB_MEMBER | STAFF | VIEWER status VARCHAR(20) DEFAULT 'ACTIVE', created_at TIMESTAMP DEFAULT NOW() ); CREATE UNIQUE INDEX ON institution_members(institution_id, user_id);
CREATE TABLE projects ( id UUID PRIMARY KEY, institution_id UUID REFERENCES institutions(id), title VARCHAR(300) NOT NULL, project_type VARCHAR(30), -- CLINICAL_TRIAL | OBSERVATIONAL | RETROSPECTIVE | -- REGISTRY | DATA_ANALYSIS | FL_STUDY | SYNTHETIC_DATA status VARCHAR(30) DEFAULT 'DRAFT', -- DRAFT | IRB_PENDING | IRB_APPROVED | RECRUITING | -- ACTIVE | DATA_LOCKED | ANALYSIS | COMPLETED pi_user_id UUID NOT NULL, -- 연구책임자 (uniqdata_db.users.id) irb_number VARCHAR(50), target_participant_count INT, required_data_categories TEXT[], -- ['VITALS','LAB_RESULT'] reward_per_participant DECIMAL(18,6), total_reward_budget DECIMAL(18,6), escrow_xrpl_tx_hash VARCHAR(64), created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE participants ( id UUID PRIMARY KEY, project_id UUID REFERENCES projects(id), user_id UUID NOT NULL, -- uniqdata_db.users.id consent_id UUID, -- uniqdata_db.consents.id status VARCHAR(20) DEFAULT 'INVITED', -- INVITED | CONSENTED | ENROLLED | ACTIVE | -- COMPLETED | WITHDRAWN | EXCLUDED reward_earned DECIMAL(18,6) DEFAULT 0, reward_paid DECIMAL(18,6) DEFAULT 0, created_at TIMESTAMP DEFAULT NOW() ); CREATE UNIQUE INDEX ON participants(project_id, user_id); CREATE TABLE project_milestones ( id UUID PRIMARY KEY, project_id UUID REFERENCES projects(id), milestone_type VARCHAR(30), -- IRB_SUBMISSION | IRB_APPROVAL | RECRUITMENT_START | -- RECRUITMENT_COMPLETE | DATA_COLLECTION | CLOSURE status VARCHAR(20) DEFAULT 'PENDING', target_date DATE, completed_at TIMESTAMP, sequence_order INT ); CREATE TABLE participation_criteria ( id UUID PRIMARY KEY, project_id UUID REFERENCES projects(id), criteria_type VARCHAR(20), -- INCLUSION | EXCLUSION field VARCHAR(50), -- "age", "gender", "chronic_conditions" operator VARCHAR(20), -- EQUALS | BETWEEN | IN | NOT_IN | GTE | LTE value JSONB ); CREATE TABLE data_requests ( id UUID PRIMARY KEY, project_id UUID REFERENCES projects(id), requester_id UUID, request_type VARCHAR(30), -- PARTICIPANT_DATA | AGGREGATE | EXPORT data_categories TEXT[], status VARCHAR(20) DEFAULT 'PENDING', core_request_id UUID, -- Core Internal API 요청 ID created_at TIMESTAMP DEFAULT NOW() );
CREATE TABLE irb_submissions ( id UUID PRIMARY KEY, project_id UUID REFERENCES projects(id), institution_id UUID REFERENCES institutions(id), submission_type VARCHAR(30), -- INITIAL | AMENDMENT | CONTINUING_REVIEW | CLOSURE status VARCHAR(30) DEFAULT 'DRAFT', -- DRAFT | SUBMITTED | PRELIMINARY_REVIEW | -- FULL_BOARD_REVIEW | REVISION_REQUESTED | -- APPROVED | CONDITIONALLY_APPROVED | REJECTED version INT DEFAULT 1, ai_generated BOOLEAN DEFAULT FALSE, ai_model_used VARCHAR(50), -- "claude-sonnet-4-5" approval_valid_until DATE, created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE irb_documents ( id UUID PRIMARY KEY, submission_id UUID REFERENCES irb_submissions(id), doc_type VARCHAR(50), -- PROTOCOL | CONSENT_FORM | INVESTIGATOR_BROCHURE | -- CASE_REPORT_FORM | AMENDMENT_REPORT | SAFETY_REPORT title VARCHAR(300), content TEXT, -- 마크다운/HTML version INT DEFAULT 1, generated_by VARCHAR(20), -- AI | MANUAL | TEMPLATE previous_version_id UUID REFERENCES irb_documents(id), created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE irb_review_comments ( id UUID PRIMARY KEY, submission_id UUID REFERENCES irb_submissions(id), reviewer_id UUID, review_type VARCHAR(20), -- PRIMARY | SECONDARY | FULL_BOARD decision VARCHAR(20), -- APPROVE | MINOR_REVISION | REJECT comment TEXT, target_document_id UUID, target_section VARCHAR(100), is_confidential BOOLEAN DEFAULT FALSE, created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE irb_workflow_history ( id UUID PRIMARY KEY, submission_id UUID REFERENCES irb_submissions(id), from_status VARCHAR(30), to_status VARCHAR(30), changed_by UUID, reason TEXT, created_at TIMESTAMP DEFAULT NOW() );
CREATE TABLE settlement_batches ( id UUID PRIMARY KEY, project_id UUID REFERENCES projects(id), batch_type VARCHAR(30), -- PARTICIPANT_REWARD | DATA_PURCHASE | FL_CONTRIBUTION status VARCHAR(20) DEFAULT 'PENDING', -- PENDING | CALCULATING | CALCULATED | -- CORE_REQUESTED | ESCROW_PROCESSING | COMPLETED total_amount DECIMAL(18,6), currency VARCHAR(10) DEFAULT 'UQD', item_count INT, escrow_tx_hash VARCHAR(64), created_at TIMESTAMP DEFAULT NOW() ); CREATE TABLE settlement_items ( id UUID PRIMARY KEY, batch_id UUID REFERENCES settlement_batches(id), recipient_user_id UUID, -- uniqdata_db.users.id recipient_wallet_address VARCHAR(60), amount DECIMAL(18,6), calculation_basis JSONB, -- {data_points_count: 150, quality_avg: 0.85, days: 90} status VARCHAR(20) DEFAULT 'PENDING', -- PENDING | PAID | FAILED | REFUNDED paid_tx_hash VARCHAR(64), paid_at TIMESTAMP ); CREATE TABLE webhook_events ( id UUID PRIMARY KEY, event_type VARCHAR(50), source VARCHAR(20) DEFAULT 'CORE', payload JSONB, status VARCHAR(20) DEFAULT 'RECEIVED', -- RECEIVED | PROCESSED | FAILED | RETRYING retry_count INT DEFAULT 0, error_message TEXT, created_at TIMESTAMP DEFAULT NOW() );

❓ FAQ
Q: 분리 기준이 "민감/비민감 데이터"인가?
A: 핵심 기준은 "환자에 대한 정보 vs 활동에 대한 정보".
사람 자체에 대한 것(건강, 신원, 키, 동의, 지갑) → Core
사람이 참여하는 활동에 대한 것(프로젝트, IRB, 정산, 기관) → UniQLab Backend
Q: uniqdata_db에는 총 몇 개 테이블?
A: 15개. users, passkey_credentials, user_profiles, user_wallets, wallet_recovery_configs, data_points, data_point_tags, data_access_logs, consents, consent_scopes, consent_audit_trail, xrpl_transactions, notifications, notification_preferences, refresh_tokens
Q: uniqlab_db에는 총 몇 개 테이블?
A: 16개 (Phase 2 포함). institutions, institution_members, projects, project_milestones, participants, participation_criteria, data_requests, irb_submissions, irb_documents, irb_review_comments, irb_workflow_history, settlement_batches, settlement_items, synthetic_datasets(Ph2), fl_rounds(Ph2), webhook_events
Q: IRB 관련 테이블은 어디에?
A: 🔵 uniqlab_db에 4개: irb_submissions, irb_documents, irb_review_comments, irb_workflow_history. IRB는 연구 "활동"이므로 UniQLab Backend 담당.
Q: 정산은 어디서?
A: 정산 계산(대상자 산정, 금액 계산, settlement_batches 생성) → 🔵 UniQLab Backend
정산 실행(XRPL EscrowFinish, UQD 크레딧 적립) → 🟢 Core
UniQLab Backend가 POST /api/v1/internal/escrow/finalize로 요청 → Core가 실행 → tx_hash 반환
Q: Internal API와 Core Push 갯수는?
A: UniQLab Backend→Core Internal API: 13개 (REST, X-Service-Auth 인증)
Core→UniQLab Backend Push API: 5개 (현재 2개 라이브, 3개 목표)

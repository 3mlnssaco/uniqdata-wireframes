## 1. 가이드라인 핵심 요약 (UniQdata 관점)

### 1.1 UniQdata의 법적 위치

UniQdata는 **B2C 건강 데이터 수집 → 가명처리 → B2B 연구자 제공** 모델로, 다음 두 가지 법적 역할을 동시에 수행합니다:

- **개인정보처리자** (PIPA 기준): 앱 사용자로부터 직접 건강 데이터를 수집하는 주체
- **인간대상연구 촉진 플랫폼**: 생명윤리법 제2조 제3호 "개인을 식별할 수 있는 정보를 이용하는 연구" (유형 3)를 지원

### 1.2 데이터 분류 체계 (3단계)

가이드라인에 따른 IRB 심의 요건 분류:

| 데이터 유형 | 정의 | IRB 요건 | UniQdata 해당 시나리오 |
|---|---|---|---|
| **익명 데이터** | 특정 개인 식별 불가 | IRB 불필요 | 공개 통계 데이터셋 활용 |
| **가명정보** | PIPA 28-2 기준 가명처리 완료 | IRB 면제 가능 (7일 fast-track) | 대부분의 B2B 데이터 제공 시나리오 |
| **식별가능 정보** | 직접/간접 식별 가능 | 정규 IRB 심의 필수 | 원본 데이터 기반 연구, 소규모 코호트 |

### 1.3 IRB 면제 Fast-Track 경로 (7일)

가명정보의 경우, 다음 조건 충족 시 **7일 이내 면제증명서 발급 가능**:

1. **개인정보처리자**(UniQdata)가 안전한 가명처리를 수행
2. **DRB**(보건의료정보심의위원회) 또는 내부 검토 위원회가 가명처리 적정성 승인
3. 연구자가 IRB에 가명처리 확인 서류 제출
4. IRB가 심의면제 해당 여부 확인 후 **7일 이내** 면제증명서 발급

### 1.4 IRB-DRB 이중 심의 구조

보건의료 데이터 활용 시 IRB와 DRB를 **통합 운영** 또는 **병행 운영** 가능:

- **통합 운영**: 하나의 위원회에서 생명윤리법 + PIPA 양쪽 심의
- **병행 운영**: IRB(생명윤리법 심의)와 DRB(PIPA 가명처리 심의)를 별도 진행 후 결과 통합

### 1.5 동의서 필수 8개 항목 (생명윤리법 제16조)

1. 연구의 목적
2. 연구 참여 기간, 절차 및 방법
3. 연구 대상자에게 **예상되는 위험 및 이익**
4. 개인정보 보호에 관한 사항
5. 동의 철회 가능 및 그 방법
6. **연구 목적 이외 사용 제한에 관한 사항**
7. **연구 관련 손해 발생 시 배상/보상**
8. 기타 (참여 중단 시 조치, 연구자 연락처 등)

### 1.6 연구계획서 필수 포함 사항 (Chapter 3)

- 연구 배경 및 목적
- 연구 대상자 선정/제외 기준
- 연구 방법론
- **데이터 출처 및 이용 적법성** (수집 근거 법조항 명시)
- **식별 가능성 평가** (가명처리 후에도 재식별 가능 여부)
- **연구 후 데이터 처리 계획** (보관/파기/반환)

### 1.7 부정적 가명처리표 (23개 데이터 항목)

가명처리 시 각 데이터 항목별 처리 방법 (완전 삭제 vs. 가명처리):

- **완전 삭제**: 성명, 주민번호, 여권번호, 외국인등록번호, 운전면허번호, 의료기관명칭, 사업자번호
- **가명처리**: 생년월일(→연령대), 주소(→시군구), 연락처(→해시), 의료기록번호(→일련번호), 진단코드(유지), 처방내역(유지), 검사결과(유지)

### 1.8 공용IRB 이용 절차 (Q9)

소속 기관이 없는 연구자를 위한 공용IRB(public.irb.or.kr) 8단계:
연구계획서 작성 → 심의신청(e-IRB) → 행정점검 → 접수(심의비 납부) → 심의(정규/신속/면제) → 연구 수행 → 과제 관리 → 종료/결과 보고

### 1.9 기관생명윤리위원회 평가·인증 (2026 상반기)

- **인증 유효기간**: 3년 (1년 연장 가능)
- **신청 자격**: 3년 이상 운영, 연 4회 이상 정기회의, 3년 누적 30건 이상 심의
- **평가 항목**: 2개 영역, 5개 범주, 40개 세부 기준
- **혜택**: 의료기관 인증 연계, R&D 사업 가산점

## 3. 필수 수정 사항 (🔴 높음)

### 3.1 irb-submission.html Step 1 수정

**현재**: 가명정보 여부 binary 선택 (예/아니오)
**변경**: 3단계 데이터 분류 + 심의 경로 분기

```
[익명 데이터] → IRB 불필요 안내 → 프로젝트 진행
[가명정보]   → DRB 승인 확인 → IRB 면제 신청 (7일 fast-track)
[식별가능]   → 정규 IRB 심의 절차 (현재 경로)
```

### 3.2 irb-submission.html Step 3 동의서 누락 항목 추가

현재 동의서 미리보기에 **3개 항목 추가** 필요:
- **"예상되는 위험 및 이익"** — 데이터 유출 위험, 건강 리포트 이익 등
- **"연구 목적 이외 사용 제한"** — 승인된 연구 범위 밖 사용 금지 명시
- **"손해 발생 시 배상/보상"** — 데이터 침해 시 보상 절차

### 3.3 irb-submission.html Step 2 필수 필드 추가

- **데이터 출처 및 적법성**: 수집 근거 법조항 (예: PIPA 15조, 17조)
- **연구 후 데이터 처리 계획**: 보관 기간, 파기 방법, 반환 절차

### 3.4 irb-status.html EXEMPTION 상태 추가

상태 흐름에 면제 경로 추가:
```
현재: DRAFT → SUBMITTED → PRELIMINARY → FULL_BOARD → APPROVED
추가: DRAFT → SUBMITTED → EXEMPTION_REVIEW → EXEMPTED (7일)
```

### 3.5 에스크로 활성화 조건 확대

```
현재: IRB status == 'APPROVED' 일 때만 에스크로 활성화
변경: IRB status IN ('APPROVED', 'EXEMPTED') 일 때 에스크로 활성화
```

📋 신속심의 (Expedited Review)

데이터 항목 추가는 기존 수집 방식의 확장이며, 참여자 위험 수준에 변화가 없어 신속심의 대상입니다.

판단 근거
  • 혈압 데이터: 웨어러블 기기로 자동 수집 → 참여자 추가 부담 최소
  • PSS-10 설문: 표준화된 설문, 소요 5분 미만 → 최소 위험
  • 연구 방법론 변경 없음 (RCT 유지)
  • 참여자 수/기간 변경 없음

2. 변경 유형

데이터 수집 항목 추가 (혈압 모니터링 + 스트레스 설문)

3. 변경 사유

혈압이 혈당-식단 상관관계 분석의 주요 교란 변수로 확인. 스트레스의 혈당 영향 보정 목적.

변화 없음. 혈압: 웨어러블 자동 수집 (비침습). PSS-10: 표준 설문 5분.

5. 동의서 변경 사항

📋 디자이너 노트: 연구 수정 + IRB 변경심의

진입 경로:
• 연구 대시보드 → "수정하기" 버튼 → 이 페이지
• 사이드바: 진행 중 연구의 서브 메뉴로 접근

변경심의 유형 (생명윤리법 시행규칙 제13조):
• 신속심의 (Expedited): 최소 위험 변경 — 데이터 추가 (비침습), 기간 연장, 보상 변경
• 정규심의 (Full Board): 중대 변경 — 방법론 변경, 대상자 기준 변경, 침습적 시술 추가
• 보고만 (Notification): 오타 수정, 연구자 연락처 변경 등 (IRB 보고만으로 처리)

핵심 플로우:
• 변경 사유 선택 → 변경 내용 diff 뷰 → AI 심의 유형 판단 → 서류 양식 생성 → 추가 에스크로 → 제출
• 변경심의 승인 전까지 기존 프로토콜대로 연구 계속 진행
• 승인 후: 프로토콜 업데이트 + 기존 참여자 재동의 + XRPL 앵커링

재동의 로직:
• 데이터/방법론 변경 → 기존 참여자 전원에게 수정 동의서 발송
• 거부 시: 기존 프로토콜대로 계속 참여 (추가 변경분만 미적용)
• 동의서 변경 → 변경 후 신규 참여자는 새 동의서로 시작

DB:
• irb_submissions.submission_type = 'amendment'
• irb_submissions에 parent_submission_id 추가 (원본 승인 참조)
• amendment_details JSON 필드: 변경 전/후 값 저장

⚙️ 기능 명세

핵심 기능:
• 진행 중 연구의 프로토콜 변경 요청
• 변경 사유 선택 (복수 선택 가능)
• 변경 전/후 비교 (diff 뷰)
• AI 기반 변경심의 유형 자동 판단 (신속/정규/보고)
• 변경심의 서류 양식 AI 자동 생성
• 추가 에스크로 예치 (보상 변경 시)
• 기존 참여자 재동의 자동화

변경심의 판단 기준 (AI):
• 데이터 추가 (비침습) → 신속심의
• 기간 연장/단축 → 신속심의
• 보상 금액 변경 → 신속심의
• 방법론 변경 → 정규심의
• 참여자 기준 변경 → 정규심의
• 침습적 시술 추가 → 정규심의
• 오타/연락처 수정 → 보고만

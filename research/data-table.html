Design Decisions & Spec Notes

1. View Mode Toggle (2 modes)

Why default to Group Comparison for RCT: The primary question in an RCT is "Is the intervention working vs control?" Researchers open this page to see between-group differences first. Individual drill-down is secondary. Non-RCT studies (observational, single-arm) would default to Individual mode.

2. Time Period Toggle (4 periods)

PeriodValue ShownDelta BasisSpecial Features
ì¼ê°„ (Daily)Today's raw valuevs YesterdayMed taken/missed, real-time alerts
ì£¼ê°„ (Weekly)7-day averagevs BaselineCompliance %, weekly trends
ì›”ê°„ (Monthly)30-day averagePercentage vs BaselineSparkline trend, monthly summary
ì‹œì ë³„ (Timepoints)BL / W2 / W4 / W8Each vs BaselineProtocol milestone view, longitudinal

3. Column Priority System (P1/P2/P3)

P1 (Red, always visible): Participant ID + cohort badge, MAGE (mg/dL), Diet Compliance (%), Status. These are the minimum columns a researcher needs to see at a glance.
P2 (Yellow, default visible): CV%, TIR (%), Medication. Important but not critical for the primary research question.
P3 (Gray, toggle to show): Blood Pressure, Sleep, Mood, HbA1c. Supplementary data that clutters the view for most use cases. Click the P3 tag to toggle visibility.

4. Delta-First Display Philosophy

Why Delta is primary: In clinical research, change from baseline/yesterday matters more than absolute values. A MAGE of 42.3 means little without context, but "decreased by 4.2" is immediately actionable. The raw value is shown in parentheses as secondary reference.
Cell format: â†“4.2 (42.3) not 42.3 â†“4.2.
Delta is 15px bold, raw is 11px gray. This 36% size difference ensures visual hierarchy.

5. Group Comparison Layout

Group Summary Bar: 3-column grid showing experiment stats, control stats, and p-values with effect size. This is the first thing a PI looks at when opening the page.
Mini Sparkline: SVG-based dual-line chart showing MAGE trends. No external library needed. Both groups rendered with distinct colors (blue=experiment, amber=control).
Table Structure: Group header rows separate experiment/control participants. Pinned average rows at the top of each group for quick reference.
p-value display: * (p < 0.05), ** (p < 0.01), *** (p < 0.001). Color-coded by significance level.

6. Individual Mode Layout

Flat table: All participants in a single sortable table (no group headers). Cohort membership shown via badge (A/B) next to participant ID.
Stats strip: Aggregate metrics (submission rate, medication compliance, threshold breaches) shown as pill-shaped chips above the table.
Alert banner: Same as group view but without group context.

7. Right Detail Panel (400px)

Shows selected participant's detail when a row is clicked. Includes today's summary cards, 7-day raw data table, baseline comparison, and participation metadata. This panel eliminates the need to navigate to a separate participant detail page for quick inspection.

8. Responsive Considerations

Below 1400px, the detail panel hides and becomes a slide-over. Column priority system ensures P1 columns remain visible even on narrow screens. P3 columns hidden by default to reduce horizontal scrolling.
